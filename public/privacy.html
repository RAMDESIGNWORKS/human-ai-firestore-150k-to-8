Privacy Policy

Last updated: March 7, 2026

This Privacy Policy describes how RMcManus Holdings LLC ("we", "us") collects, stores, and uses data within the Executive Assistant application ("Application"). This Application is an internal operational tool restricted to authorized RMcManus Holdings LLC personnel.

Data We Collect

Authentication data:

  • Google account email address and display name (via Firebase Authentication)
  • Firebase user ID (UID) used to scope all stored data

Google OAuth tokens (when Google is connected):

  • OAuth 2.0 access token and refresh token for Google Calendar, Gmail, and Google Contacts
  • These tokens are stored encrypted in Google Cloud Secret Manager or Firestore, accessible only to the Application's backend functions

Application data:

  • Chat session history and AI conversation transcripts
  • Tasks, to-do items, and completion records
  • Memory facts and notes saved by the AI assistant
  • Push notification device tokens (FCM tokens) for task reminders

Operational data:

  • Desktop automation job queue entries and results
  • Machine index data from the local operations worker (file paths, installed software index)
  • Session metadata (timestamps, session IDs)

How We Use This Data

  • To authenticate authorized users and maintain session state
  • To provide AI-assisted executive assistant features (scheduling, drafting, task management)
  • To access Google Calendar, Gmail, and Contacts on the authorized user's behalf
  • To deliver push notifications for task reminders and alerts
  • To route and execute approved desktop automation workflows
  • To persist memory, tasks, and context across sessions

Third-Party Services

The following third-party services process data as part of normal Application operation:

  • Google Firebase / Google Cloud Platform — authentication, Firestore database, Cloud Functions, Firebase Hosting, Secret Manager, and Cloud Messaging
  • OpenAI — AI language model processing. Chat messages and tool context are sent to OpenAI's API for response generation. See OpenAI's Privacy Policy.
  • Google APIs — Calendar, Gmail, and Contacts APIs are accessed using stored OAuth tokens on behalf of the authorized user

No data is shared with advertising networks, data brokers, or any third party beyond those listed above.

Google API Data Use Disclosure

This Application's use of data received from Google APIs, including Calendar, Gmail, and Contacts, is limited to providing executive assistant functionality to the authorized user. This data is not used to train AI models, is not shared with third parties other than OpenAI (solely for generating responses), and is not used for advertising purposes. This Application complies with the Google API Services User Data Policy, including the Limited Use requirements.

Data Storage and Security

  • All data is stored on Google Cloud (Firebase) infrastructure in the us-central1 region
  • Data is encrypted in transit via TLS/HTTPS and encrypted at rest by Google Cloud's default encryption
  • OAuth tokens are stored in Firestore with Firestore Security Rules restricting access to the authenticated owner only
  • Access to backend functions is restricted via Firebase Authentication token verification

Data Retention

  • Chat sessions and tasks are retained indefinitely until manually deleted by the user
  • Google OAuth tokens are refreshed automatically and remain until the user revokes access
  • Push notification tokens are retained until the device token changes or is explicitly removed

Revoking Google Access

You may revoke the Application's access to your Google account at any time through your Google Account Permissions page. Removing access will disable Calendar, Gmail, and Contacts features in the Application.

Your Rights

  • As an authorized user, you have full access to and ownership of your data stored within the Application
  • Data can be deleted by contacting an administrator of the Firebase project or directly through the Firestore console

Changes to This Policy

This policy may be updated as the Application evolves. The "Last updated" date above reflects the most recent revision.
